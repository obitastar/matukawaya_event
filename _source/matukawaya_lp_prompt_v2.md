## 1. プロジェクト概要

### 1.1 何を作るか

**老舗呉服店「御福まつかわや」が年に二度開催する、着物展示会のランディングページ（LP）** を1ページの単体HTMLで作成します。

### 1.2 主目的

**広告 → LP着地 → 来場予約フォーム送信** までの導線を完成させること。
副目的：ブランドの世界観を伝え、ブランド認知を高める。

### 1.3 流入チャネル

以下の3経路からの流入を想定：

1. **Web広告**（Google・Meta） — スマホ流入多め
2. **Instagram・Facebookのオーガニック投稿** — 既存ファン層、スマホ流入ほぼ100%
3. **チラシ・DMの二次元コード** — 40〜60代の着物愛好家、スマホでQRを読み込んで来訪

→ **モバイルファースト必須**。デスクトップは美しく、スマホは確実に読めて確実に予約できる、を両立させる。

### 1.4 ターゲット

幅広い層（伝統に興味がある人全般）。優先順位：
- 第一：**40〜60代の着物愛好家**（既存顧客・潜在的な購入層）
- 第二：**20〜30代の若年層**（浴衣・入門層、SNS経由）

→ 文字サイズ・コントラスト・タップ領域は**40代以上にとっても快適**であることを優先。装飾的すぎて読みづらいのは厳禁。

### 1.5 コピーライティングの方針（最重要）

このLPの言葉は、**詠む**ように書きます。説明的・販促的な表現は徹底的に避けてください。

**書くときの原則：**

- 短く、深く。一文は20〜35字程度を基本とする
- 体言止め・倒置・余白を恐れない
- 動詞を抜き、名詞と助詞だけで語れるなら、そう書く
- 「美しい」「素晴らしい」「特別な」のような**形容詞での称賛は使わない**。代わりに、具体的な事象・数字・固有名詞で読み手を動かす
- 価格訴求は明示的にはしない。ただし「永く付き合える着物に、ふさわしい出会い方を」のように、**質と納得感が両立することを匂わせる**表現は許容する
- 「皆様」「お客様」のような呼びかけは最小限に。読み手と1対1で対話する

**やってはいけない例：**
- × 「素晴らしい着物を、特別なお値段でご提供！」
- × 「日本一お得な着物展示会、ぜひお越しください！」
- × 「皆様のご来場を心よりお待ちしております」

**良い例：**
- ○ 「永く着るほど、馴染んでいく一枚を。」
- ○ 「京都と江戸の手仕事が、上本町に集う二日間。」
- ○ 「次の一枚と、出会いに来てください。」

「日本一安く高級着物を提供する」というまつかわやの信念は、LP上で明示的には謳いません。ただし、**「永く付き合える着物との、ふさわしい出会い方」**という言い回しで、品質と納得感の両立をそっと示唆します。

## 2. デザイン仕様

### 2.1 デザインの方向性

**和モダン × 陰翳礼讃 × ミニマル**

- **薄暗い背景に反物・着物をドラマチックに浮かび上がらせる**ビジュアルが主役
- 茶室の静謐な空気感を写真とテキストで再現
- 装飾は最小限。**写真と余白で語る**
- 文字情報は明朝体の縦組みを軸に、英字でジャンプ率を作る
- 「老舗の格」と「現代の洗練」の両立 — 古臭くも軽くもならない

### 2.2 カラーパレット

```css
:root {
  /* ベース：陰翳の世界観 */
  --color-bg: #0F0E0C;          /* 漆黒に近い墨黒（メインBG） */
  --color-bg-alt: #1A1815;      /* やや明るい墨黒（セクション切替用） */
  --color-paper: #F5F1E8;       /* 生成り（明るいセクション・余白用） */
  --color-paper-warm: #EBE5D6;  /* 生成り(やや濃) */

  /* テキスト */
  --color-text-light: #E8E2D2;  /* 暗い背景上の本文 */
  --color-text-mute: #A89E8B;   /* 暗い背景上の補助 */
  --color-text-dark: #1F1C16;   /* 明るい背景上の本文 */
  --color-text-dark-mute: #6B6356; /* 明るい背景上の補助 */

  /* アクセント */
  --color-accent: #8B1A1A;       /* 朱赤（CTA・要所のみ、使い過ぎ注意） */
  --color-accent-hover: #A02020;
  --color-gold: #B8956A;         /* 金茶（罫線・装飾・西原染匠系で使用） */

  /* 罫線 */
  --color-border-dark: #2D2A24;
  --color-border-light: #D8D2C2;
}
```

**配色の使い方ルール：**
- LP全体は**墨黒ベース**を基調にする（FV、企画紹介、会場紹介など主要セクションは暗背景）
- 「呼吸」を入れたいセクション（イントロ、開催概要、フォーム）は**生成りベース**で切り替え
- 朱赤（accent）は**CTAボタンと「200年」などの数字強調のみ**。乱用厳禁
- 金茶（gold）は**装飾罫線**と**西原染匠セクションの差し色**に限定

### 2.3 タイポグラフィ

#### フォント指定

```css
/* 日本語 明朝 */
font-family: "Shippori Mincho", "Noto Serif JP", "游明朝", "Yu Mincho", "ヒラギノ明朝 ProN", serif;

/* 英字 セリフ */
font-family: "Cormorant Garamond", "Playfair Display", Georgia, serif;

/* 補助：本文・UI用ゴシック */
font-family: "Noto Sans JP", "游ゴシック", "Yu Gothic", "ヒラギノ角ゴ ProN", sans-serif;
```

Google Fontsから読み込む（最小限のweightのみ）：
- `Shippori Mincho` (weight: 400, 500)
- `Noto Sans JP` (weight: 400, 500)
- `Cormorant Garamond` (weight: 300, 400 italic)

#### スケール（モバイル → デスクトップで可変）

| 用途 | モバイル | デスクトップ | weight | family |
|---|---|---|---|---|
| FV大見出し | 32px | 64px〜80px | 500 | Shippori Mincho |
| セクション見出し | 28px | 48px | 500 | Shippori Mincho |
| 数字強調（200年など） | 56px | 120px | 300 | Cormorant Garamond |
| 英字小見出し | 14px | 16px | 400 italic | Cormorant Garamond |
| 本文 | 15px | 17px | 400 | Shippori Mincho |
| キャプション | 12px | 13px | 400 | Noto Sans JP |
| ボタン | 15px | 16px | 500 | Noto Sans JP |

#### タイポルール

- **行間**：本文 1.9〜2.0、見出し 1.4〜1.5（和文は広めに取る）
- **文字間**：見出しは `letter-spacing: 0.1em` 以上、本文は `0.05em`
- **縦組み**：FVのキャッチコピー、セクション見出しの一部で `writing-mode: vertical-rl` を活用
- **句読点・約物**：自然な日本語組版を意識

### 2.4 余白・グリッド

- **コンテンツ最大幅**：1200px（FVのみ100vw）
- **セクション縦余白**：モバイル80px、デスクトップ160px（広め推奨、詰め込まない）
- **テキストブロック幅**：本文は最大640px、見出しはこの限りでない
- **写真と次の要素の間**：必ず40px以上の余白

### 2.5 写真の見せ方

**最重要セクション。** 写真がコンテンツの主役なので、見せ方の質がLPの質を決める。

#### 基本ルール

- **薄暗い背景に反物・着物をドラマチックに配置**するのが基本トーン
- 写真は**画面いっぱい近くまで使う**（小さくチマチマ並べない）
- 写真の周囲に十分な余白を確保（写真同士を隣接させない）
- 写真の上にテキストを重ねる場合、可読性のための**グラデーションオーバーレイ**を必ず付与

#### 推奨する写真配置パターン

1. **フルブリードFV**：100vw × 70〜90vh、画面下部からのグラデーションでコピーを浮かび上がらせる
2. **片寄せレイアウト**：左に写真（55%）、右にテキスト（45%）。次のセクションで左右反転
3. **縦長写真の中央配置**：反物をドレープさせた縦写真を中央に、両脇は十分な黒背景
4. **横スクロールギャラリー**：制作工程・柄ディテールなど、複数の写真を見せたいとき

#### Lazy loading

- ファーストビュー以外の画像は **`loading="lazy"`** を必ず付与
- `<img>` には `width` `height` 属性を必ず付ける（CLS対策）

### 2.6 アニメーション・インタラクション

**控えめに、上品に。** 派手な動きは老舗の格を損なう。

- **スクロール時のフェードイン**：要素が画面に入ったら 800ms かけて opacity 0→1、translateY 20px→0
  - `IntersectionObserver` で実装
  - `prefers-reduced-motion: reduce` を尊重して、有効な場合はアニメーションをオフ
- **ホバー時**：CTAボタンは背景色の変化のみ。装飾的な動きは不要
- **FV写真**：軽いズームイン（10秒で 100% → 105%）を `prefers-reduced-motion` に応じてオン/オフ
- **横スクロールギャラリー**：スワイプ可能、スナップポイント付き

### 2.7 レスポンシブ ブレークポイント

```css
/* モバイル（基準） 〜 640px */
/* タブレット @media (min-width: 641px) */
/* デスクトップ @media (min-width: 1024px) */
/* 大画面 @media (min-width: 1440px) */
```

モバイルファーストで記述すること。

## 3. ページ構成（セクション別の詳細指示）

全11セクション + 固定ヘッダー + モバイル固定下部CTA。

**重要：以下に記載するコピーは、そのまま実装に使用する完成稿です。** 勝手な変更・追記・削除は行わないでください。文言の調整が必要な場合は質問してください。

### 固定ヘッダー（全セクション共通）

- **位置**：`position: fixed; top: 0;`
- **構成**：左に「御福 まつかわや」のテキストロゴ（Shippori Mincho）、右に「来場予約」のCTAボタン
- **背景**：スクロール量0px時は完全透明、50px以上スクロールすると半透明の墨黒背景（`rgba(15, 14, 12, 0.85)` + `backdrop-filter: blur(8px)`）
- **高さ**：モバイル56px、デスクトップ72px
- **モバイル時**：右の「来場予約」ボタンはアイコン or 短いテキスト「予約」にして圧迫感を減らす

### モバイル固定下部CTAバー

- **位置**：`position: fixed; bottom: 0;`、モバイルのみ表示
- **構成**：「来場予約をする」の大きなボタン1つ。タップでフォームへスムーズスクロール
- **表示タイミング**：FV を抜けたあとから表示（最初の100vh以降で `transform: translateY(0)` でスライドイン）

## 4. 技術要件

### 4.1 ファイル構成

単体HTMLでシンプルに公開できる構成。

```
matukawaya-event/
├── index.html             # メインHTML
├── assets/
│   ├── css/
│   │   └── style.css      # メインCSS（HTMLからリンク）
│   ├── js/
│   │   └── main.js        # IntersectionObserver、固定CTA制御等
│   ├── img/
│   │   ├── fv.jpg                    # FV画像（仮）
│   │   ├── nishihara-main.jpg        # 西原染匠メイン
│   │   ├── nishihara-process-01.jpg  # 工程1
│   │   ├── ...
│   │   ├── chikusen-main.jpg
│   │   ├── chikusen-detail-01.jpg
│   │   ├── ...
│   │   ├── matukawaya-portrait.jpg   # 八代目
│   │   ├── kohroan.jpg               # 茶室
│   │   └── og-image.jpg              # OGP用 1200x630
│   └── favicon/
│       ├── favicon.ico
│       └── apple-touch-icon.png
└── README.md              # 公開手順、画像差し替え手順を記載
```

CSSとJSはHTMLにインライン化せず、外部ファイルにすること（保守性のため）。

### 4.2 仮画像の扱い

- **すべての画像はプレースホルダー**として扱う
- 仮画像はUnsplashなどの和素材で**それっぽいもの**を配置するか、`<img>` の `src` に空のグラデーション背景（CSSで生成）を設定
- 各画像の `<img>` タグの直前に **HTMLコメント** で「ここに差し替える画像の指示」を必ず記述：
  ```html
  <!-- IMAGE: FV画像。薄暗い背景に反物がドラマチックに浮かぶ写真。1920x1080以上推奨。差し替え先：assets/img/fv.jpg -->
  <img src="assets/img/fv.jpg" alt="..." width="1920" height="1080">
  ```
- README.mdに **画像差し替え手順の一覧** を必ず記載

### 4.3 フォント読み込み

- Google Fontsを `<link>` で読み込み
- `font-display: swap` を確実に効かせる
- 読み込むweightは指示通り最小限に（不要な太さは含めない）

### 4.6 アクセシビリティ

- すべての `<img>` に意味ある `alt` 属性
- 装飾画像は `alt=""`
- `<button>` と `<a>` を意味通りに使い分ける
- フォーム要素には `<label>` を必ず関連付け
- カラーコントラストはWCAG AA準拠（特に墨黒背景上のテキスト色）
- `prefers-reduced-motion: reduce` を尊重
- キーボード操作で全機能にアクセス可能
- フォーカスインジケータを和モダンな見た目で実装（朱赤の細枠など）

### 4.7 パフォーマンス

- **Lighthouse Performance スコア 90+ を目標**
- 画像は次の方針：
  - WebP形式を優先、フォールバックでJPG
  - `loading="lazy"` をFV以外に付与
  - `width` `height` 属性を必ず指定（CLS対策）
- CSS・JSは最小限。フレームワーク不使用。Vanilla JSのみ。
- フォントはサブセット化を検討（読み込み遅延を回避）

### 4.8 ブラウザ対応

- iOS Safari 最新2バージョン
- Android Chrome 最新2バージョン
- PCの Chrome / Edge / Safari / Firefox 最新版
- IE11非対応で問題なし
